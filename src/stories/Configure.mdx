# Nexus Gaming Component Library

Welcome to the Nexus Gaming component documentation. This library contains a collection of React components designed for building gaming-related web applications.

## Component Organization

Our components are organized in specific categories:

### Common Components (`src/components/common/`)
Reusable UI components used throughout the application:
- **ColorDot** - Visual status indicator with customizable colors
- **DateDisplay** - Formatted date display with multiple format options
- **Hero** - Featured content banner with image background and badges
- **LoadingOverlay** - Full-screen loading indicator with animation
- **NotificationIcon** - Icon system for different notification types
- **Notifier** - Toast notification system
- **SideColorBadge** - Container with vertical accent stripe

### Article Components (`src/components/articles/`)
Components for article display and management:
- **ArticleCard** - Card display for gaming articles
- **ArticleCardSkeleton** - Loading placeholder for article cards

### Game Components (`src/components/games/`)
Components specific to game-related features:
- **GameCard** - Display card for game information
- **GameCardSkeleton** - Loading placeholder for game cards

### Layout Components (`src/components/layout/`)
Navigation and structure components:
- **AppLayout** - Main application layout
- **Navbar** - Top navigation bar
- **AdminMenu** - Administration navigation menu
- **UserAvatar** - User profile display

## Theme Support

All components support multiple themes through our Material-UI configuration:

- **Light** (default) - Clean, professional light theme
- **Dark** - Eye-friendly dark mode
- **Gaming** - Gaming-inspired color scheme
- **Retro** - Retro gaming aesthetic

You can switch themes in the Storybook toolbar to preview components in different themes.

## Development Guidelines

### Creating Stories

Each component should have its own stories file (`Component.stories.tsx`) with:

1. Default story showing basic usage
2. Stories demonstrating key variations
3. Interactive controls when applicable
4. Proper TypeScript typing

### File Structure
```plain
src/
  components/
    category/
      Component.tsx
      Component.stories.tsx
      ComponentSkeleton.tsx (if applicable)
```

### Story Organization

Stories are organized by component category in the Storybook sidebar. Each component's stories should follow this pattern:

```typescript
// Example story structure
const meta = {
  title: 'Category/ComponentName',
  component: Component,
  tags: ['autodocs'],
  parameters: {
    layout: 'centered', // or 'fullscreen' for larger components
  },
};

// Basic usage
export const Default = { ... };

// Variations
export const WithCustomization = { ... };
export const LoadingState = { ... };
```

## Common Patterns

### Using Article Components

```tsx
// Basic ArticleCard usage
<ArticleCard
  article={{
    title: "Latest Gaming News",
    category: { name: "News" },
    publishedAt: "2025-04-26T10:00:00Z",
    // Additional properties...
  }}
/>

// With games
<ArticleCard
  article={{
    title: "Game Comparison",
    category: { name: "Reviews" },
    games: [
      { id: "1", title: "Game One" },
      { id: "2", title: "Game Two" }
    ],
    // Additional properties...
  }}
/>
```

### Loading States

We provide skeleton components for loading states:

```tsx
// During content loading
<ArticleCardSkeleton />
<GameCardSkeleton />
```

### Notifications

Using the notification system:

```tsx
const { showSuccess, showError } = useNotifier();

// Show success notification
showSuccess('Operation completed successfully');

// Show error notification
showError('An error occurred');
```

## Contributing

When adding new components:

2. Add comprehensive stories demonstrating all features
3. Include proper documentation in the stories
4. Ensure theme compatibility
5. Add loading states if applicable
